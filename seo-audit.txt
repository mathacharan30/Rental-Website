Remaining Issues — Prioritised Action Plan
🔴 CRITICAL — Fix Within 1 Week
C1 — The client:only="react" architecture makes all service page body text invisible to Google

This is the single biggest ranking blocker on the site. Every service landing page (all 10) renders its H1, intro paragraphs, highlights, FAQs, and pricing entirely inside React — Google sees a blank body until JavaScript executes. Google AI Overviews, Perplexity, and ChatGPT do not execute JavaScript at all.

What to do: For each service .astro page, render the static content directly as Astro HTML before the <ReactApp>. The data is already in serviceLandingData.js — it just needs to move from React to Astro output. Only keep React for interactive behaviour (accordion open/close).

Minimal approach per service page:


---
import { servicePages } from '../data/serviceLandingData'
const data = servicePages['rental-clothing-mysuru']
---
<BaseLayout ...>
  <!-- Server-rendered, crawler-visible content -->
  <main>
    <h1>{data.h1}</h1>
    {data.introParagraphs.map(p => <p>{p}</p>)}
  </main>
  <!-- React handles navigation, cart, interactive UI -->
  <ReactApp client:load />
</BaseLayout>
Note: Change client:only → client:load so the React app also hydrates from the server-rendered HTML rather than replacing it.

Files: all 10 service .astro pages.

🟠 HIGH — Fix Within 2 Weeks
H1 — No H1 in server-rendered HTML on the homepage

The hero <h2> is loaded from a CMS API call in React. Google has no visible heading to associate with the homepage's keyword target. Add a static <h1> in index.astro before <ReactApp>:


<h1 class="sr-only">Bridal Wear on Rent in Mysuru &amp; Bangalore — People &amp; Style</h1>
Using sr-only keeps it visually hidden but fully crawlable.

H2 — No AggregateRating schema anywhere on the site

Star ratings in Google SERPs are the highest-CTR schema for local service businesses. The testimonials component already loads real reviews from the database — this data needs to flow into structured data.

What to do: Once you know your Google Maps rating and review count (check your GBP dashboard), add this to the defaultSchema in BaseLayout.astro:


"aggregateRating": {
  "@type": "AggregateRating",
  "ratingValue": "4.8",
  "reviewCount": "47",
  "bestRating": "5"
}
H3 — No BreadcrumbList schema on any page

Breadcrumb trails in SERPs improve click-through and are absent site-wide. Add to each service page .astro file:


{
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Home", "item": "https://peopleandstyle.in" },
    { "@type": "ListItem", "position": 2, "name": "Bridal Clothing Rental Mysuru", "item": "https://peopleandstyle.in/rental-clothing-mysuru" }
  ]
}
H4 — No named artists or photographers anywhere on the site

For queries like "bridal makeup artist Mysuru" and "wedding photographer Mysuru", Google rewards pages with named, identifiable humans. The team section in AboutUs.jsx is commented out. Uncomment it, add a real name and bio. This is the largest single E-E-A-T gap.

H5 — Photography pages: change schema @type to PhotographyBusiness

photography-services-mysuru.astro and bangalore use ProfessionalService — the correct Schema.org type is PhotographyBusiness. A one-line change per file.

H6 — Deploy before Google re-crawls the 4 new pages in the sitemap

🟡 MEDIUM — Fix Within 1 Month
M1 — FAQ schema only covers 5 of 17 questions

The faq.astro FAQPage schema has 5 questions but the React FAQ component has 17. The other 12 are invisible to crawlers. Add the remaining questions to the schema in faq.astro.

M2 — FAQ schema answers are too short for AI citation (30–55 words; need 100–150)

Expand each FAQ schema answer to be self-contained and ~100–150 words. Include brand name, city, specific pricing, and a closing CTA. This directly improves Google AI Overview and Perplexity citation probability.

M3 — Product schema: change itemCondition from UsedCondition

ProductDetail.jsx sets itemCondition: "https://schema.org/UsedCondition". For rental items, omit this field or use RefurbishedCondition. Also add offers.seller and offers.priceValidUntil.

M4 — Add streetAddress to all PostalAddress blocks

Schema address blocks have city/region/country but no street address. This weakens Local Pack eligibility. Add the studio's actual street address.

M5 — Contact form uses mailto: fallback

The contact form in ContactUs.jsx uses window.location.href = 'mailto:...' which silently fails on mobile or if no email client is configured. Replace with a proper API POST.

M6 — Update llms.txt with pricing and service detail

llms.txt is too sparse. Add pricing ranges (₹300–₹12,000), the booking workflow steps, and the cancellation policy. This is the primary source AI assistants use to describe the business.

M7 — Gallery alt text is generic ("Gallery 1", "Gallery 2")

Update to descriptive alt text: "Bridal lehenga rental in red and gold at Mysuru — People & Style". Each image should describe the outfit, colour, and occasion.
